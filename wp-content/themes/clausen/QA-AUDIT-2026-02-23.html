Critical Findings

Page: /join/  ·  All conversion CTAs across the site

Critical #1 — /join/ page is completely empty

The single most important conversion page on the site contains only a title and a footer. The following links all point here and hit a dead end:

  • Hero “Join us” button
  • “Membership” nav link
  • All three homepage tier buttons (“Join as individual/family/student”)
  • Bottom CTA “Join TCLAS”
  • Resources page “Join TCLAS” button

The actual PMPro membership levels table lives at /membership-levels/, but nothing routes visitors there.

Suggested fix: Either add the [pmpro_levels] or [pmpro_checkout] shortcode to the /join/ page content, or redirect all CTA links to /membership-levels/. Ideally, build out /join/ with a branded checkout experience.

Critical #2 — /contact/ page is completely empty

Linked from the footer in two places (“Contact” in the Organisation column and “Contact” in the bottom bar). The page renders only a title — no contact form, no email address, no phone number, no mailing address.

Suggested fix: Add a contact form (Gravity Forms, WPForms, or simple mailto: link), an email address, and optionally a mailing address.

Component: Primary navigation  ·  Header

Critical #3 — Nav link labels are swapped

“Membership” in the main nav → /join/ (the empty page). The red “Join” button → /membership-levels/ (a raw PMPro table). These are backwards. A visitor clicks “Membership” expecting to learn about membership and sees nothing; they click “Join” expecting checkout and get an unstyled data table.

Suggested fix: In WordPress Admin → Appearance → Menus, swap the URLs: “Membership” should point to the membership info/levels page, and “Join” should point to the checkout/signup flow. Or unify both under a single well-built /join/ page.

High-Priority Findings

Component: Header brand area  ·  header.php lines 39–51

High #4 — Header brand link creates 3 orphan links (broken HTML)

the_custom_logo() outputs its own <a> tag wrapping the logo image. Since it’s nested inside the <a class="tclas-brand"> wrapper, the browser auto-corrects the invalid nesting by splitting the outer link into empty fragments. The rendered DOM contains:

  • An empty <a class="tclas-brand"> (no content, no accessible name)
  • A second empty <a class="tclas-brand"> fragment inside .tclas-brand__logo
  • The actual <a class="custom-logo-link"> with the logo image
  • The brand name and tagline text orphaned outside any link
Suggested fix: Replace the_custom_logo() with direct <img> output using wp_get_attachment_image() on the custom logo ID. This avoids the nested <a> problem entirely.

Page: Homepage  ·  Sections 3 and 5

High #6 — Large empty illustration placeholders on homepage

The “About TCLAS” section and “Membership” section both render large light-blue rectangles (~350px tall) where illustrations should be. On mobile, the About placeholder takes nearly the full viewport height. These make the site look unfinished to first-time visitors.

Suggested fix: Either upload illustrations via the ACF theme options panel, or hide the placeholder divs until images are available by adding .tclas-welcome__illustration:empty, .tclas-membership__illustration:empty { display: none; }

Files: page.php, front-page.php  ·  All generic pages

High #7 — container vs. container-tclas inconsistency

Every section in front-page.php and both wrappers in page.php use Bootstrap’s .container, while the project convention is .container-tclas (max-width: 1200px). This causes subtle width differences between the homepage/generic pages and pages like Events, About, and Resources that use the correct class.

Suggested fix: Replace class="container" with class="container-tclas" in page.php (lines 21, 38) and all sections in front-page.php.

Pages: Events list, single event  ·  TEC date format settings

Medium #14 — Date/time format doesn’t follow AP style guide

Event cards show “March 7 @ 6:30 pm - 8:30 pm”. Per the style guide, this should read “March 7 at 6:30 p.m.” (no “@”; periods in a.m./p.m.; en-dash for ranges). This is governed by The Events Calendar’s date format settings in WordPress admin.

Suggested fix: In WP Admin → Events → Settings → Display, update the time format to use “g:i a” and customize the separator. May need a filter hook for the “@” → “at” replacement and the period-style a.m./p.m.

Accessibility Findings

Component: Header brand  ·  header.php

A11y #19 — Two empty brand links announced by screen readers

Per finding #4, the broken <a class="tclas-brand"> elements have zero text content. Screen readers will announce “link” with no accessible name twice in succession before reaching the actual logo link. Confusing for keyboard/screen reader users.

Suggested fix: Fix the nested link issue (finding #4). As a quick interim fix, add aria-hidden="true" tabindex="-1" to the empty fragments.

Your notes

Pages: Homepage  ·  Footer  ·  Template strings

A11y #20 — Luxembourgish strings missing lang="lb" in templates

“Wëllkomm” (homepage h2), “Mir sinn hei” (hero eyebrow + footer tagline) are not wrapped in <span lang="lb">. The ltz-tagger module only processes the_content and the_excerpt filters — hardcoded template strings are not covered. Screen readers will attempt to pronounce these as English.

Suggested fix: Manually wrap each Luxembourgish string in template files with <span lang="lb">, or extend the ltz-tagger to also filter the_title and custom template hooks.

Your notes

Pages: Homepage sections 3 and 5

A11y #21 — Illustration placeholders may confuse assistive tech

tclas_illustration() receives alt text (e.g., “TCLAS members gathered around a table”) but renders an empty <div> with no image. If the alt text is applied as an aria-label or role="img", screen readers may describe a nonexistent image.

Suggested fix: If no image is uploaded, output nothing (or output the <div> with aria-hidden="true" and no alt text).
